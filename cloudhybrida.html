Cloud Hybrida

Esta plataforma le permite al cliente utilizar los recursos disponibles para crear varias modalidades de cloud. Cloud Hybrida se encarga de la instalación, configuración y administración de todos los recursos necesarios para mantener una cloud. Nuestra plataforma es agnóstica al hardware que se utiliza lo cual permite al cliente utilizar desde servidores de alto rendimiento hasta computadoras personales y laptops para crear una cloud.

Cloud Hybrida cuenta con varias modalidades:

Cloud Privada



A diferencia de una Cloud Publica donde los recursos se encuentran en el centro de datos de el proveedor sin acceso, las Cloud Privadas se encuentran localizadas en locaciónes de el cliente ya sea su centro de datos, sus oficinas o algún otra locación donde este mantenga su hardware. La Cloud Privada, como su nombre lo indica, solo permite el uso de los recursos de quien la esta administrando y no es de uso publico.

Una Cloud Privada puede usarse como base para crear una Cloud Publica y ofrecer los servicios ofrecidos por una Cloud Privada al publico.

La mayoría de las cloud privadas tienen sus caso de uso en los siguientes sectores:

  • Educación para la administración de recursos de computo para laboratorios, desarrollo e investigación o enseñanza.
  • Financiero para la administración de recursos de computo dentro de lugares bajo alta seguridad, hospedar aplicaciónes y sistemas en ambientes PCI, sistemas soportando transacciónes bancarias, FinTech, acciónes y valores.
  • Telecomunicaciónes para la administración de servidores que hospedan aplicaciónes y sistemas de telefonía, crear Cloud Publicas, servicios de hospedaje, sitios web, aplicaciónes IoT y 5G.
  • Medios y Entretenimiento permitiendo la administración de recursos TI utilizados en la creación de animaciónes, videojuegos, clusters de aplicaciónes, sistemas web, entre otros.
  • Sector Privado de Tecnología de Información creando clouds privadas a la empresa para hospedar sistemas de recursos humanos, sistemas financieros, CRM, aplicaciónes y sistemas web específicos a la empresa, análisis de datos, consumo e ingreso de información y métricas de sistemas externos, reducir costos de utilizar cloud publicas.
  • Gobierno para aplicaciónes militares, jurídicas y de sistemas internos en linea.


Cloud Híbrida



La modalidad cloud híbrida como su nombre lo implica permite tener una o mas combinaciónes cloud bajo una misma administración, ya sea

  • Cloud Privada - Cloud Publica
  • Cloud Publica - Otra Cloud Publica
  • Cloud Privada - Cloud Privada


Esta modalidad permite la mayor flexibilidad de el ecosistema permitiendo al cliente utilizar los recursos ya sean de uno o mas proveedores Cloud o multiples instalaciónes de la modalidad Privada de nuestra plataforma con la finalidad de reducir costos, requisitos de localidad, hacer uso de características especificas al proveedor Cloud, entre otros. Conexiones entre una y mas Clouds son realizadas por enlaces dedicados en el cual la transferencia de datos se da encriptada o por el uso de redes privadas virtuales (VPN’s).

Una de las mayores ventajas de una cloud híbrida es la facilidad de mover sistemas y aplicaciónes de una cloud a otra, con la finalidad de incrementar el procesamiento, agilizar la ejecución, reducir costos instantáneos, distribuir el sistema a nuevas locaciónes y acercar el sistema a usuarios de el cliente o simplemente aumentar o reducir la capacidad de el sistema.

Cloud Edge



La modalidad cloud edge de la plataforma Cloud Híbrida esta diseñado para todas aquellas aplicaciónes y sistemas que necesitan ser distribuidas o locales a donde se recolecta la información que procesan.

Principales casos de uso son:

  • Internet de las Cosas (IoT)
  • Puntos de Venta
  • Oficinas satélite
  • Telefonia 5G


La ventaja de Cloud Edge ante los otros tipos de modalidad de cloud es la distribución de datos y de carga de trabajo, en Cloud Publicas y Privadas todo el manejo y procesamiento de datos se hace en un lugar central, usualmente un centro de datos, que aun contando con alta disponibilidad, incurre en costos de transferencia de datos desde la fuente (origen de datos) hasta el destino (cloud). En el caso de Cloud Edge todo el procesamiento se realiza en nodos distribuidos que se encuentran localizados en el mismo lugar donde los datos se están generando, reduciendo la necesidad de transferencia de datos así como su costo; una ves procesados los datos, el resultado es enviado a la parte central de esta modalidad Cloud para abastecerla y permitir compartir los resultados con otros sistemas.

Cloud Edge es considerado el nuevo tipo de computación en nube en el cual todos los dispositivos logran tener un acceso casi instantáneo a los datos de un sistema y otros sistemas o aplicaciónes reduciendo los costos aun mas e incrementando la velocidad de acceso.
